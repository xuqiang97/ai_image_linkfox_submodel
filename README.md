## 1. 本期目标

当前 AI 生图系统调用 LinkFox 生图接口时，系统固定使用 `provider=BANANA_PRO`，分辨率未显式传参，由 LinkFox 默认按 2K 生成。

现 LinkFox 已支持多个子模型及不同算力档位。为方便业务按生成效果、算力成本和使用场景选择模型，本期需在 AI 生图任务创建链路中新增 **LinkFox子模型** 选择能力。

本需求属于 **LinkFox 内部 provider 选择能力扩展**，不新增一级模型，不改变现有 **LinkFox、自研、阿里绘蛙** 的一级模型结构，也不纳入多模型并行矩阵列。

## 3. 页面与交互规则

### 3.1 字段位置与展示

在新增任务抽屉、编辑任务抽屉、查看任务抽屉中新增字段：

**LinkFox子模型（查看官方算力说明）：**

字段位于 **「模型与数量配置」区域上方紧邻展示**，并在新增任务抽屉、编辑任务抽屉、查看任务抽屉中始终展示。

批量导入任务时，在批量导入配置区域支持统一选择 LinkFox子模型，字段同样位于「模型与数量配置」区域上方紧邻展示。

该字段在任务抽屉和批量导入配置区域始终展示；当 LinkFox 计划生成张数 = 0 时，仅保存配置，不参与实际生成传参。字段展示不代表实际调用 LinkFox；仅当 LinkFox 计划生成张数 > 0 时，该字段才参与 LinkFox 建任务传参。

字段下方展示蓝色提示，提示样式复用现有系统蓝底提示样式，包含感叹号图标，**多模型适配** 加粗：

> **多模型适配**：该字段仅 LinkFox 使用；自研、阿里绘蛙忽略此字段。

字段下方展示灰字说明：

> 图片分辨率固定按2K生成，算力按2K单张算力展示仅供参考，实际以官方最新说明为准

“查看官方算力说明”点击后新标签页打开：

https://open.ziniao.com/docSupport?docId=176

### 3.2 可编辑态与只读态

| 场景     | 展示规则                                           |
| ------ | ---------------------------------------------- |
| 新增任务抽屉 | 展示 LinkFox子模型下拉，可编辑                            |
| 编辑任务抽屉 | 仅未提交 / 草稿态任务可进入，字段可编辑                          |
| 查看任务抽屉 | 已提交、生成中、审核中、已完成任务通过“查看任务”入口进入，字段复用下拉组件置灰展示不可点击 |
| 批量导入任务 | 在批量导入配置区域支持统一选择 LinkFox子模型                     |

### 3.3 L3 沉浸式大屏展示

| 当前图片来源              | 顶部标题区 Tag 展示          |
| ------------------- | --------------------- |
| LinkFox + 谷歌香蕉1Pro  | `LinkFox` `谷歌香蕉1Pro`  |
| LinkFox + GPT图片2中品质 | `LinkFox` `GPT图片2中品质` |
| 自研                  | `自研`                  |
| 阿里绘蛙                | `阿里绘蛙`                |

规则：

1. 当前图片为 LinkFox 图片时，顶部标题区展示两个同色模型 Tag：`LinkFox` + 当前图片实际使用的 LinkFox子模型。
2. 当前图片为自研图片时，顶部标题区仅展示 `自研` Tag。
3. 当前图片为阿里绘蛙图片时，顶部标题区仅展示 `阿里绘蛙` Tag。
4. 历史 LinkFox 图片未记录子模型字段时，L3 按 `LinkFox` + `谷歌香蕉1Pro` 展示。
5. 切换不同图片时，顶部模型 Tag 和 LinkFox子模型 Tag 随当前图片自动刷新。

## 4. LinkFox子模型选项

字段类型：下拉单选。

无缓存或缓存不可用时，系统默认选项为：**谷歌香蕉1Pro**

| optionCode            | 页面展示名称                | provider 传参 | quality 传参 | resolution 传参 | 2K 算力 / 张 |
| --------------------- | --------------------- | ----------- | ---------- | ------------- | --------- |
| BANANA_PRO_2K         | 谷歌香蕉1Pro（40算力/张）      | BANANA_PRO  | 不传         | 2K            | 40        |
| BANANA_2_2K           | 谷歌香蕉2（40算力/张）         | BANANA_2    | 不传         | 2K            | 40        |
| GPT_2_IMAGE_LOW_2K    | GPT图片2低品质（15算力/张）     | GPT_2_IMAGE | low        | 2K            | 15        |
| GPT_2_IMAGE_MEDIUM_2K | GPT图片2中品质（50算力/张）     | GPT_2_IMAGE | medium     | 2K            | 50        |
| GPT_2_IMAGE_HIGH_2K   | GPT图片2高品质（150算力/张）    | GPT_2_IMAGE | high       | 2K            | 150       |
| AIDRAW_EDIT_2K        | LinkFox自研模型V1（20算力/张） | AIDRAW_EDIT | 不传         | 2K            | 20        |
| WAN2_7_2K             | 万相2.7（25算力/张）         | WAN2_7      | 不传         | 2K            | 25        |

说明：

1. `optionCode` 用于前端下拉选项识别和前端缓存，不作为 LinkFox 接口传参字段。
2. “LinkFox自研模型V1”仅表示 LinkFox 接口 `provider=AIDRAW_EDIT` 的子模型选项，不等同于系统一级模型中的“自研模型”。
3. GPT图片2低品质、GPT图片2中品质、GPT图片2高品质在页面上作为三个独立选项展示，接口 `provider` 均传 `GPT_2_IMAGE`，通过 `quality` 区分。
4. 非 GPT图片2 的 LinkFox子模型，接口不传 `quality` 参数。
5. 系统统一显式传 `resolution=2K`，前端不提供 1K、4K 选择。
6. 页面选项仅展示模型名称和算力，不展示 provider、quality、resolution 等接口传参细节。
7. 页面算力按当前 LinkFox 官方说明展示；后续官方规则调整时，需同步更新页面展示和 PRD 选项。

## 5. 默认值与缓存规则

| 场景                                        | 规则                   |
| ----------------------------------------- | -------------------- |
| 首次进入新增任务抽屉、批量导入弹窗                         | 默认选中谷歌香蕉1Pro         |
| 新增任务成功，且 LinkFox 计划生成张数 > 0               | 前端缓存本次选择的 LinkFox子模型 |
| 批量导入成功，且至少存在 1 个 LinkFox 计划生成张数 > 0 的成功任务 | 前端缓存本次选择的 LinkFox子模型 |
| LinkFox 计划生成张数 = 0                        | 不更新缓存                |
| 用户仅切换下拉但未成功创建任务                           | 不更新缓存                |
| 创建任务失败、批量导入全部失败                           | 不更新缓存                |
| 下次进入同版页面                                  | 优先使用缓存值作为默认值         |
| 美工版、运营版                                   | 分别缓存                 |
| 不同登录用户                                    | 分别缓存                 |
| 缓存值不存在、下线、不可选                             | 回退为谷歌香蕉1Pro          |
| 编辑任务、查看任务、复制重跑、批量复制重跑                     | 以任务已保存配置为准，不使用缓存值覆盖  |

前端缓存值必须使用 `optionCode`，不得仅缓存 `provider`。

用户默认偏好仅前端缓存，不需要入库。实际任务使用的 LinkFox子模型必须入库。

前端缓存仅用于新增任务抽屉、批量导入弹窗的默认选中值，不影响已保存任务的查看、编辑、复制重跑、批量复制重跑配置。

缓存 key 需区分登录用户、美工版/运营版、缓存版本，建议结构为：

`ai_image_linkfox_sub_model_{userId}_{bizType}_v1`

## 6. 功能与接口规则

### 6.1 新增、批量导入、编辑、查看

| 场景                 | 规则                                         |
| ------------------ | ------------------------------------------ |
| 新增任务               | 可选择 LinkFox子模型                             |
| 批量导入任务             | 可统一选择 LinkFox子模型，应用于本次导入创建的任务              |
| 未提交 / 草稿态任务        | 通过编辑任务抽屉修改 LinkFox子模型                      |
| 已提交、生成中、审核中、已完成任务  | 通过查看任务抽屉只读查看 LinkFox子模型                    |
| LinkFox 计划生成张数 > 0 | 按所选子模型传递 `provider`、`quality`、`resolution` |
| LinkFox 计划生成张数 = 0 | 字段仅展示和保存配置，不参与实际生成传参                       |

本文中的“LinkFox 计划生成张数”指当前任务中 LinkFox 列下所有图片类型计划张数合计。

**LinkFox子模型为任务级配置，一个任务仅选择一个 LinkFox子模型，作用于该任务内所有 LinkFox 计划生成图片；本期不支持按图片类型或单张图片分别选择 LinkFox子模型。**

当 LinkFox 计划生成张数 = 0 时，仅保存任务配置；不创建 LinkFox 生成明细和图片结果，BI 生成与效果分析不计入该子模型。

查看任务时，只读展示任务实际使用的 LinkFox子模型。

| 实际配置                                    | 展示文案                    |
| --------------------------------------- | ----------------------- |
| `provider=BANANA_PRO`                   | LinkFox · 谷歌香蕉1Pro      |
| `provider=GPT_2_IMAGE`，`quality=medium` | LinkFox · GPT图片2中品质     |
| `provider=AIDRAW_EDIT`                  | LinkFox · LinkFox自研模型V1 |

### 6.2 复制重跑、批量复制重跑

复制重跑、批量复制重跑任务时，必须继承原任务的 LinkFox子模型配置：

| 字段         | 规则    |
| ---------- | ----- |
| provider   | 继承原任务 |
| quality    | 继承原任务 |
| resolution | 继承原任务 |

复制重跑、批量复制重跑不得使用前端缓存值覆盖原任务配置。

### 6.3 批量修改

本期不支持批量修改 LinkFox子模型。

### 6.4 接口传参

调用 LinkFox 建任务接口时：

| 参数         | 规则                                                 |
| ---------- | -------------------------------------------------- |
| provider   | 按用户选择的 LinkFox子模型传值                                |
| quality    | 仅 GPT图片2 传 `low`、`medium`、`high`；其他 provider 不传该参数 |
| resolution | 统一显式传 `2K`                                         |

| 页面选项          | provider 传参 | quality 传参 | resolution 传参 |
| ------------- | ----------- | ---------- | ------------- |
| 谷歌香蕉1Pro      | BANANA_PRO  | 不传         | 2K            |
| 谷歌香蕉2         | BANANA_2    | 不传         | 2K            |
| GPT图片2低品质     | GPT_2_IMAGE | low        | 2K            |
| GPT图片2中品质     | GPT_2_IMAGE | medium     | 2K            |
| GPT图片2高品质     | GPT_2_IMAGE | high       | 2K            |
| LinkFox自研模型V1 | AIDRAW_EDIT | 不传         | 2K            |
| 万相2.7         | WAN2_7      | 不传         | 2K            |

自研、阿里绘蛙生成链路不读取、不传递 LinkFox子模型字段。

## 7. 数据记录与 BI 同步要求

本期不仅是页面新增“LinkFox子模型”下拉，还需保证后续可追溯、可统计、可分析。系统需在任务配置、生成过程、图片结果和 BI 同步链路中记录实际使用的 LinkFox子模型信息。

| 记录范围  | 业务要求                                                    |
| ----- | ------------------------------------------------------- |
| 任务配置  | 记录任务选择的 LinkFox子模型、GPT图片2品质、分辨率，用于任务查看、编辑、复制重跑、批量复制重跑继承 |
| 生成过程  | 记录每次 LinkFox 生成实际使用的子模型信息，用于统计不同子模型的生成成功率、失败率和失败原因      |
| 图片结果  | 记录每张 LinkFox 图片实际使用的子模型信息，用于 L3 展示、审核结果、上传使用和上线后业务效果分析  |
| BI 同步 | BI 库和 T+1 同步链路需同步上述数据，支持 BI 看板按 LinkFox子模型维度细拆分析        |

数据记录口径：GPT图片2 记录对应 quality 值；非 GPT图片2 的 LinkFox子模型，quality 记录 NULL；resolution 统一记录 2K。

LinkFox子模型信息仅适用于 LinkFox 生成链路；自研、阿里绘蛙不参与该字段分析。

后续 BI 看板需支持按 LinkFox子模型分析生成效果、审核效果、使用效果和业务效果。其中，生成成功率、失败率以生成过程记录为准；审核、上传、上线后业务效果以每张图片实际使用的 LinkFox子模型为准。

## 8. 历史兼容、权限与风险

### 8.1 历史任务兼容

历史 LinkFox 任务、历史 LinkFox 生成明细、历史 LinkFox 图片结果未记录 `provider`、`quality`、`resolution` 时，统一按以下口径处理：

| 字段         | 默认处理       |
| ---------- | ---------- |
| provider   | BANANA_PRO |
| quality    | NULL       |
| resolution | 2K         |

历史任务查看、L3 沉浸式大屏查看、复制重跑、批量复制重跑、BI 分析均按上述默认口径处理。

### 8.2 权限说明

本需求不新增菜单权限，不新增按钮权限。沿用现有 AI 生图相关页面和操作权限。

本需求不改变美工版、运营版既有数据权限范围。

### 8.3 风险与边界

| 风险           | 说明                               | 应对策略                                                |
| ------------ | -------------------------------- | --------------------------------------------------- |
| 模型概念混淆       | LinkFox子模型可能被误解为一级模型             | 明确其仅为 LinkFox 内部 provider，不改变一级模型结构                 |
| GPT图片2三档无法区分 | 低品质、中品质、高品质共用 provider           | 必须保存 quality，前端缓存必须使用 optionCode                    |
| 前端缓存误覆盖      | 缓存值可能误影响已有任务                     | 缓存仅用于新增任务、批量导入任务默认值，已有任务以保存配置为准                     |
| 算力展示过期       | LinkFox 官方算力规则后续可能调整             | 页面标注仅供参考，并以官方最新说明为准；前端按最新文档更新展示文案                   |
| 子模型选项变更      | LinkFox 后续可能新增、下线或调整子模型          | 本期选项按当前 PRD 固定；后续如有变更，需同步更新前端字典、PRD 选项和缓存失效规则       |
| 历史任务字段缺失     | 老任务无 provider、quality、resolution | 按 provider=BANANA_PRO、quality=NULL、resolution=2K 处理 |
| BI 数据缺口      | 未同步生成过程、图片结果和 BI 链路              | 任务配置、生成过程、图片结果、BI 同步链路需同步记录                         |

## 9. 验收标准

页面展示以 HTML 原型为准，重点验收以下内容：

| 模块    | 验收重点                                                             |
| ----- | ---------------------------------------------------------------- |
| 页面展示  | 任务抽屉字段位置、批量导入弹窗字段位置、标题、蓝色提示、灰字说明、下拉选项、查看态置灰展示、L3 顶部 Tag          |
| 功能操作  | 新增、批量导入、编辑、查看、复制重跑、批量复制重跑、批量修改不支持                                |
| 接口传参  | provider、quality、resolution 按规则传递；非 GPT图片2 不传 quality            |
| 数据记录  | 任务配置、生成过程、图片结果均能记录实际使用的 LinkFox子模型信息                             |
| BI 同步 | BI 库和 T+1 同步链路支持新增数据，BI 看板可按 LinkFox子模型维度细拆分析                    |
| 历史兼容  | 历史 LinkFox 数据按 provider=BANANA_PRO、quality=NULL、resolution=2K 兼容 |
| 前端缓存  | 使用 optionCode，按用户和版本分别缓存，不覆盖已有任务配置                               |

5. 字段下方展示蓝色提示：**多模型适配**：该字段仅 LinkFox 使用；自研、阿里绘蛙忽略此字段。
6. 字段下方展示灰字说明：图片分辨率固定按2K生成，算力按2K单张算力展示仅供参考，实际以官方最新说明为准。
7. 下拉选项名称、顺序、算力展示正确。
8. “查看官方算力说明”支持新标签页打开，链接地址为：https://open.ziniao.com/docSupport?docId=176
9. 查看态复用下拉组件置灰展示不可点击。
10. 任务列表页、L2 审核工作台本期不展示 LinkFox子模型。
11. L3 顶部标题区展示规则正确：LinkFox 图片展示 `LinkFox` + 当前 LinkFox子模型两个同色 Tag；自研、阿里绘蛙图片仅展示对应一级模型 Tag。
12. 页面字段位置、提示样式、查看态置灰状态、L3 Tag 样式需与 HTML 原型保持一致。

### 9.2 功能与接口验收

1. 新增任务、批量导入任务可选择 LinkFox子模型。
2. 未提交 / 草稿态任务可通过编辑任务抽屉修改 LinkFox子模型。
3. 已提交、生成中、审核中、已完成任务仅可通过查看任务抽屉只读查看 LinkFox子模型。
4. 批量修改不支持修改 LinkFox子模型。
5. 复制重跑、批量复制重跑可继承原任务的 LinkFox子模型配置。
6. LinkFox 计划生成张数 > 0 时，按所选子模型参与接口传参。
7. LinkFox 计划生成张数 = 0 时，字段仅展示和保存配置，不参与实际生成传参，不创建 LinkFox 生成明细，不产生 LinkFox 图片结果记录。
8. 7 个 LinkFox子模型选项均需覆盖提交验收，确认 `provider`、`quality`、`resolution` 传参正确。
9. GPT图片2 三个选项对应的 `quality` 正确。
10. 非 GPT图片2 的 LinkFox子模型不传 `quality` 参数。
11. 所有 LinkFox子模型均显式传 `resolution=2K`。
12. 自研、阿里绘蛙生成链路不读取、不传递 LinkFox子模型字段。

### 9.3 数据、BI、缓存验收

1. 任务配置、生成过程、图片结果均能记录实际使用的 LinkFox子模型信息。
2. 历史任务、历史生成明细、历史图片结果按 `provider=BANANA_PRO`、`quality=NULL`、`resolution=2K` 正确兼容。
3. BI 库和 T+1 同步链路支持新增数据，BI 看板可按 LinkFox子模型维度细拆分析。
4. 用户首次进入新增任务抽屉、批量导入弹窗时，默认谷歌香蕉1Pro。
5. 前端缓存值使用 `optionCode`，不得仅缓存 `provider`。
6. 新增任务成功且 LinkFox 计划生成张数 > 0 后，前端缓存本次选择。
7. 批量导入成功且至少存在 1 个 LinkFox 计划生成张数 > 0 的成功任务后，前端缓存本次选择。
8. LinkFox 计划生成张数 = 0、用户仅切换下拉但未成功创建任务、创建任务失败、批量导入全部失败时，不更新缓存。
9. 编辑任务、查看任务、复制重跑、批量复制重跑均以任务已保存配置为准，不使用缓存值覆盖。
10. 美工版、运营版缓存互相独立，不同用户缓存互相独立。
11. 缓存值不存在、下线、不可选时，回退为谷歌香蕉1Pro。
